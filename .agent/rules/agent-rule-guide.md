## Context Navigation & Governance Protocol

### Phase 1: Context Routing

* **Root Analysis:** Always start by analyzing the root `./AGENTS.md` to identify the active domain (e.g., Backend, UI, Database).
* **Nested Context:** If the task falls into a specific path (e.g., `./app/api/AGENTS.md`), prioritize the rules in that sub-file over generic knowledge.

### Phase 2: Execution Compliance

* **Immutable Rules:** Strictly adhere to the **Immutable** and **Do's & Don'ts** defined in the active `AGENTS.md`.
* **No Fluff:** Keep responses concise. No emojis allowed.

### Phase 3: Self-Correction

* **Validation:** Before finalizing, audit the generated code against the active rules. Correct any contradictions immediately.
